2. 1주차 주요 개념 키워드 적어보기
  -버전관리 : 프로젝트 상태가 변경되는 정보를 알고 있다는 것. Git은 가장 널리 쓰이는 버전관리 도구로 commit을 사용  해 관리.

  - git 초기화 : 컴퓨터에 있는 프로젝트를 git이 관리하는 프로젝트로 만들기
  - commit :
    - 현재 프로젝트의 상태를 찰칵 저장하기
    - 누가, 언제 commit 했는지 정보와 프로젝트 변경 내용
    - 작업내역이 어떤 것인지 알아볼 수 있게 적는 메시지들 'commit 메시지'
  - add: / staging, 스테이팅 : commit에 반영할지 안할지는 파일 단위로 선택 가능. Commit에 반영할 파일을 선택하는 것
  - commit history : commit 한 내역, 역사
  - repo : git으로 관리되는 프로젝트를 git에서는 repo라고 부름.
  - 내 컴퓨터에 저장되어있는 리포지토리를 로컬 repo
  - GitHub 처럼 다른 곳에서 접속할 수 있는 공간에 저장되어있는 것을 원격 repo
  - push : 로컬 repo 외 commit 들을 원격 repo에 반영하기 (push)!
